# 名詞介紹

下方是一棵很典型的樹

圓圈部分稱為節點，藍線部分則稱為分枝
紅色節點稱為樹根，樹根可以視為所有節點的祖先，黃色節點則稱為終點節點或樹葉節點。 (上) 與子節點 (下)。樹中某節點的高度則表示此節點至樹葉節點的最路長路徑，舉例來說紅色節點的高度就是 3 而樹的高度也是 3 ，深度則是樹根至該節點的高度，上方圖中最左邊的黃色節點深度是 3。高度是由節點出發到樹葉，深度則是由樹根出發到節點。接度表示樹中的世代關係，一代為一個階度，上方的階度為 4

# 二元樹

一般的樹可以有很多分支，這樣會造成紀錄上的不便，於是我們規範出二元樹，二元樹是由節點所組成的有限集合，對每個節點而言最多只會有兩個分支，具有以下特性

  • 二元樹的節點各樹可以是零，而一般樹至少由一個節點組成
  • 二元樹有排列順序的關係，而一般樹則沒有
  • 二元樹每個節點的最多分支就是 2

下方三種都是二元樹

最左邊的樹是每一個節點都只有左子樹，中間的樹若是按照先填樹根、左子樹、右子樹的順序那就稱為完整二元樹，若如最右邊的情況，那就稱為滿枝二元樹。接著來看一些二元樹的現象

  1. 一棵二元樹的 iii 階度最多有 2i−12^{i-1}2i−1 的節點
  2. 一棵二元樹若 n0n_0n0​ 表示所有樹葉節點 (終點節點)，n2n_2n2​ 表示所有分支度為 2 的節點，那會得到 n0=n2+1n_0=n_2+1n0​=n2​+1

我來解釋一下第二點，假設全部節點樹為 nnn ，因為二元樹的特性，這些節點必是 0 個分支、1 個分支、2 個分支，我們用 n0、n1、n2n_0、n_1、n_2n0​、n1​、n2​ 表示，可以得到 n=n0+n1+n2n=n_0+n_1+n_2n=n0​+n1​+n2​ 如果用分枝數 (B) 表示節點數，則可以得到 n=分枝數(B)+1n=分枝數 (B)+1n=分枝數(B)+1 這個加一是因為樹根節點無法用分枝表示，又 B=n1+2×n2B=n_1+2 \times n_2B=n1​+2×n2​ 整理一下可以得到 n0=n2+1n_0=n_2+1n0​=n2​+1
若一棵二元樹總共有 n 個節點，目前在第 i 個節點，若 2×i>n2\times i >n2×i>n 表示這個節點沒有左子樹，若 2×n+1>n2\times n+1 >n2×n+1>n 表示這個節點沒有右子樹

# 節點的追蹤

追蹤樹狀結構時我們經常使用前序追蹤 (preoder)、後序追蹤 (postoder)、中序追蹤 (inorder) 這三種，前序追蹤是指先追蹤樹根、左節點、右節點，中序追蹤是指先追蹤左節點、樹根、右節點，後序追蹤則是左節點、右節點、樹根，總之是依照樹根的位置決定追蹤的名稱
下方圖片演示了在不同追蹤演算法下的情況

我們來看程式碼

二元樹的追蹤也可以用非遞迴方式完成，這時候就可以使用迴圈 + stack
雖然說的輕鬆，但我想好久，preorder 的演算法如下

  1. 建立兩個堆疊
  2. 把樹根放入堆疊 1
  3. 把堆疊 1 的第一項 pop ，push 到堆疊 2
  4. 若有左、右子節點把左子節點、右子節點放入堆疊 1
  5. pop 出堆疊 2 的第一項
  6. 重複 3~5 直到堆疊 1 空了

需要注意的是左子節點、右子節點放入堆疊的順序，在 preorder 中左子節點應該會先被印出，因為他們較早成為樹根，所以應該先放入右子節點再放左子節點，左子節點才會在上
面。

Inorder traversal (中序遍歷) 比較簡單，會先拜訪左子節點，再拜訪父節點，最後拜訪右子節點。我們先拜訪所有左子節點途中經過的父節點們就先存在一個 stack 裡面，等到沒有更多左子節點時，就把 stack 中的父節點 pop 並拜訪其右子節點。這裡特別注意一下 return 成立在沒有左子節點且堆疊中沒有東西 (沒有父節點)。

postorder 的基本演算法和 preorder 類似，流程如下

  1. 先把樹根放入 stack 1
  2. 把 stack 1 的第一項 pop 並放入 stack 2
  3. 若有左子節點、右子節點放入 stack 1
  4. 重複 2、3
  5. 當 stack 1 空時，依序 pop 出 stack 2 的每一項

一樣要注意左右子節點放入的順序，先放左再放右，因為 stack 的特性是先進後出！
如果還是覺得有點困難，這裡有動態的追蹤情況

  • 前序追蹤 preorder

  • 中序追蹤 inorder

  • 後序追蹤 postorder
