概要

在 C++ 中有一個函數叫做 next_permutation ，它的作用是可以依照字典排序法列出一個數列 ( 或字串 ) 的下一個排列，所謂的字典排列就是先比較第一個數，第一個數一樣大時( ASCII 碼相同 )，比較第二個數，直到分出大小，舉例來說 1 2 3 這三個數字的下一個排列是 1 3 2 ，再下一個是 2 1 3，接著 2 3 1、3 1 2、3 2 1，那我們要怎麼用 C 來實現呢？
先來看看 C++的

// next_permutation example
#include <iostream>     // std::cout
#include <algorithm>    // std::next_permutation, std::sort

int main () {
  int myints[] = {1,2,3};

  std::sort (myints,myints+3);

  std::cout << "The 3! possible permutations with 3 elements:\n";
  do {
    std::cout << myints[0] << ' ' << myints[1] << ' ' << myints[2] << '\n';
  } while ( std::next_permutation(myints,myints+3) );

  std::cout << "After loop: " << myints[0] << ' ' << myints[1] << ' ' << myints[2] << '\n';

  return 0;
}

出來的結果是這樣的

咦陣列在函數中不是傳址嗎？怎麼迴圈結束最後跑出來還是1 2 3？這是因為 next_permutation() 這個函數的回傳值是一個布林值，如果還找得到比呼叫時還要大的排列，就把陣列變成比較大的排列然後回傳 true，如果找不到，就由小到大排列該陣列，然後回傳 false

手刻一個

既然 C 沒有，那我們就手刻一個 C 的 next_permutation 函數，在這裡我盡量做得和 C++ 的功能相同，首先要先來看看到底怎麼找出下一個排列的，有四步驟

  1. 由右至左找到第一個比其後項小的數，把該項設為 flag_index
  2. 由右至左找到第一個比 flag_index 所指的數大的數，交換他們兩個
  3. 把 flag_index 後的數由小到大排列
  4. 當 flag_index 小於0時，return false

舉例來說，尋找下面這個例子的 next_permutation

把 index6 ( 此刻數值為 4 ) 設為 flag_index，因為 4<7

由右至左找到第一個比 4 大的數，發現是 6，交換他們倆個

由小到大排列 6 之後的數

這樣就找到了 5 9 3 1 2 8 6 4 7，是 5 9 3 1 2 8 4 7 6 的下一個排列

接下來看程式碼，先看比較簡單的，不是用指標做的

#define swap(a,b){int x=b;b=a;a=x;}
#include<stdio.h>
#include<stdbool.h>
bool nper(int num[],int length){
    int i,j,flag_index;
    for(i=length-1;num[i-1]>=num[i];i--);
    flag_index=i-1;
    if(flag_index>=0){
        for(i=length-1;num[i]<=num[flag_index];i--);
        swap(num[flag_index],num[i]);
        for(i=flag_index+1,j=length-1;i<j;i++,j--){
            swap(num[i],num[j]);
        }
        return true;
    }
    else{
        for(i=0,j=length-1;i<j;i++,j--){
            swap(num[i],num[j]);
        }
        return false;
    }
}

把要排的陣列和長度(個數)傳進函數，然後依照上面列出的 4 步驟進行，比較需要注意的是由小到大排序，這裡看起來沒有和排序有關的程式碼呀只有不斷的 swap (交換)，為甚麼會這樣？解答這個問題我們先來看看甚麼時候會用到排序， 2 種情況

  • 陣列由大到小排列，無法找到下一個排列
    既然陣列由大到小排，要由小到大排就是倒過來排，所以是不斷的頭尾交換
  • 從flag_index 之後到結尾
    做完第二步之後可以發現，flag_index 之後到結尾也是由大到小排列，所以想要變成由小到大排也只需要不斷的頭尾交換

在上面這個函數中如果想要排列字串，記得把參數型態改一下，如果只想排列某幾個數字記得把長度改一下
接著看一下用指標做的

#define swap(a,b){int x=b;b=a;a=x;}
#include<stdio.h>
#include<stdbool.h>
bool nper(int* start, int* end);
int main(){
    int num[9]={5,9,3,1,2,8,4,7,6};
    nper(num,num+8);//注意這裡傳的是位址
    for(int k=0;k<9;++k){
        printf("%d ",num[k]);
    }
    printf("\n");
    return 0;

}
bool nper(int* start, int* end){
    int *p,*flag_index,*q;
    for(p=end;(*(p-1))>=(*p);p--);
    flag_index=p-1;
    if((flag_index-start)>=0){
        for(p=end;(*p)<=(*flag_index);p--);
        swap(*flag_index,*p);
        for(p=flag_index+1,q=end;p<q;p++,q--){
            swap(*p,*q);
        }
        return true;
    }
    else{
        for(p=start,q=end;p<q;p++,q--){
            swap(*p,*q);
        }
        return false;
    }
}

結論

在 C++ 提供的函數中不只可以排數字陣列也可以排字串，所以我想他的參數應該是 void* 型態(才能兩者都接)，但這也意味著函數內部要有一些步驟去確認傳進來的參數型態，才能決定要用怎麼樣的指標做後續處理，這部分有點麻煩也不是 next_permutation 的重點，所以在這裡依然使用 int* 的型態。我們自己手刻時可以為自己的情況量身打造只要寫出能滿足自己需求的函數就好，但寫函數庫裡的函數就必須考量到很多使用者的不同需求，函數的適用範圍越廣越好，因此產生了以上差異
另外除了 next_permutation，還有prev_permutation (前一個排列)，步驟和上面說得很類似，只要把小改成大就可以了，步驟如下

  1. 由右至左找到第一個比其後項大的數，把該項設為 flag_index
  2. 由右至左找到第一個比 flag_index 所指的數小的數，交換他們兩個
  3. 把 flag_index 後的數由大到小排列
  4. 當 flag_index 小於0時，return false
