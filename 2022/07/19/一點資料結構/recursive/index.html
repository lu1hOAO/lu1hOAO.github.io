# 概要

一個呼叫本身的函數稱為遞迴 (recursive)，遞迴通常拿來處理可以被分成多個有限步驟處理的問題，而且每個處理步驟都很相似，所以我們就可以先做一點點，然後交給下一棒，每一棒都做一點點，最後就會有做完的一天。因為遞迴具有上述的特性，因此在使用遞迴時比須注意以下幾點

  • 函數要有 return 條件
  • 每次呼叫都要處理掉部分問題，recursive 的核心在於每步都做一點點，而不是丟給後面做
  • 觀察 return 後是否需改變數值，產生新的遞迴條件
  • 思索非遞迴解，比較其時間與空間複雜度

# 河內塔

十九世紀在歐洲有一款遊戲稱為河內塔 (Towers of Hanoi)。此遊戲是一位法國數學家發明。遊戲設定是這樣的：有 64 個大小不同的金盤子，另有三根柱子為 A、B、C ，今想把 64 個金盤子從 A 柱移到 C 柱，過程中一次只能移動一個盤子，且大的盤子不能放在小的盤子之上，但可以借助 B 柱的幫忙，現在希望利用程式印出每一步的狀況。
這個題目在高中機率與統計真的超常出現，但大部分都是在問走幾步，知名度大概僅次於庭院深深深幾許吧哈哈！
河內塔的演算法如下

  1. 搬移 n-1 個盤子從 A 至 B
  2. 搬移第 n 個盤子從 A 至 C
  3. 搬移 n-1 個盤子從 B 至 C

# n 個皇后

在概要部分有說到做遞迴時需要考慮 return 之後要不要改變變數數值來產生新的遞迴，這種情況通常發生在具有「選擇」時，因為要考慮同一步的不同選擇 (比如是或否)，所以一開始先假設某一種結果 (比如是) 走完之後再回來把選擇改成否，看看改成否會造成什麼樣的結果，這樣具有回溯性的遞迴，很常被用在動態規劃，可以幫助我們模擬多種情況，以下就用 n 個皇后的例子說明。
n 個皇后是指在一個 n×nn\times nn×n 的方格中填入 n 個皇后，皇后之間不可以在同一 row、同一 column ，也不可以在同一對角線上，在這前提下我們要計算有幾組解，以四個皇后為例，在 4×44\times 44×4 的方格中填入 4 個皇后總共有幾組解，並列印出這些情況
在開始寫程式碼之前，我們先來做一點分析，由於皇后不可以在同一 row 上，所以對結果來說一定是剛好每一個 row 都只有一個 queen ，我們可以把在 row 0 的皇后視為 0 號 皇后，在 row 1 的皇后視為 1 號皇后，所以現在對每一個皇后我們只需要去找他們正確的 column 位置就好了。透過這項分析減少變數是很重要的一步。
接下我們要判斷同一 column 、對角線上是否有其他皇后，我在這裡採的方法是建立一個 col 陣列，col [0] 表示零號皇后的 column 位置，如果今天 i 號皇后和 j 號皇后在同一 column 上，那 col[i]==col[j] ，若 i 號皇后和 j 號皇后在同一對角線上，那 col [i] - col [ j ] = | i - j |
下方 check 函數的程式碼就是在執行這段，最後皇后位置以 1 表示，空位則以 0 表示

這段程式碼其實還有很多值得分析的地方，像是填寫的時候是從 0 開始填，所以在 check 函數中只需考慮比目前編號還小的皇后、當 return 回來後要把原本設定的清除，用來產生新遞迴。
在遞迴中也要注意方向，是從目前往前遞迴 (比如費氏數列)，還是從目前往後遞迴 ( 比如 combination )，這會牽涉到迴圈的判斷是設定，所以在最時務必自己仔細地想過一遍，不要隨便抄 code 喔！
以上就是 recursive 的介紹，算是有點進階卻很重要的章節吧！
